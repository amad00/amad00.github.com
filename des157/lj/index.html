Game UX Best Practices

May 18, 2021

In Game UX Best Practices, Amir Dori talks about best practices for mobile game design that helps with user experience. There are few things to consider such as UI Positions, Sliders, Pop-Ups, and more. Every design should focus on keeping the user engaged and wanting stay on the app. If there are pop-ups in which users have to make decisions, the positive or good decisions that are good for your game should be placed on the right. Pop-ups, which should be activitated by user interactions, should also avoid the "X" design choice as users may instantly close the pop-up. The close button should also appear after a short delay so users read the content before immediately closing it. Presenters (game characters) are also a good design choice, especially when it comes to "rate us" and "store" dialogs, where the looking that the user. All these best practices not only benefits the user, but you as the game designer as well, as it keeps the users on your app.

Best Practices for Modals

April 22, 2021

Modals (also known as overlays or dialogs) is content that sits on top of the main page. User's are meant to interact with it before being able to interact with the main page once more. Modals are great for when you want to grab the user's attention, need user input, when you need to show addiction information in content (as well as not in context). Modals are appropriate in these situations but they have become overused so much that users automatically dismiss these windows when they pop up.

An effect and appropriate modal window should have some key features:

  • Users should be able to close the model, which can be done through a button, keyboard press, or clicking outside of the window
  • It should have a descriptive title
  • Buttons should be name appropriately and should be actionable
  • The modal should not be very large, and should not use more than 50% of the page
  • Use a Lightbox effect when a modal is opened. This will darken the background and bring attention to what you want the user to focus on, the modal
  • Modals should be initiated by users and should not pop up automatically
  • Avoid using modals on mobile device as the screen is too small and the content becomes difficult to see as well as navigate

Best Practices for Form Design

April 15, 2021

Implementing best practices and ideas for form design helps to improve user experiences. These practices include ideas such as knowing how to format and size input fields, proximity of labels with their input fields, and minimizing massive amounts information and breaking it down into multiple steps to help prevent cognitive overload. Feedback is also great for user experience. Feedback can include having a focus on the current input field the user is on, and displaying error messages if their input invalid or incomplete.

Feedback provides users with clear notice of whether they have filled in the inputs correctly and their form has been submitted or that they need to make some changes. I personally find feedback very helpful when filling out forms as well as actions that prevent cognitive overload. A majority of the forms that I have filled were usually long with no indication of how long it is. This discourages me as user from going through that experience again.

An online form that I have used was creating an account on Starbucks website. You are able to see how long the form is, and it is very short. It uses one column layout and doesn't mark the mandatory fields with asterisks. On certain fields, once you click on it, it provides more information about what the field is, or what is needed in that field. If the input field is not filled out, it provides feedback by highlighting the section in red and bringing up an error message. All the input fields are labeled and you also have the option to see your password. I thought this was a great example of implementing best practices for forms.
